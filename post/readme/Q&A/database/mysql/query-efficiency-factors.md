```md

```
  
## SQL

## 索引
* 索引数量
```md
  MySQL 优化器需要扫描索引，以确定哪些索引可以应用到当前查询。
```

* 索引数据量（数据记录数）
```md
  二级索引会消耗大量的空间，另外索引中的主键查找可能效率更慢（看算法，如果是Hash 可能影响较小），
  二级索引肯定会受影响。
```

* 索引类型
```md
  通常情况下，覆盖索引的效率较高，主键索引（聚簇索引）可以直接索引记录，在某些排序场景也可以使用，
  二级索引，需要两次查询。
```

* 索引列的选择
```md
  主键尽量简单，否者二级索引的存储会增大。
  根据常用查询和关键查询来选择索引列。
```

* 索引列顺序
```md
  影响多列索引是否可用。
```

* 前缀索引前缀长度
```md
  根据索引的选择性来确定前缀索引的长度。
```
